# Teaching Materials — AI Agent Guide

> 一对一教培备课资料仓库，存储所有学生各周的教学材料。
> **任何修改后必须同步更新 Teaching-System/AGENTS.md 的变更日志。**

## 系统定位

本仓库是教学备课系统的**数据输出仓库**，AI 备课助手按规范生成的所有材料都写入这里。
配套仓库见 [Teaching-System/AGENTS.md](https://github.com/1686756626/Teaching-System/blob/main/AGENTS.md)。
前端展示网站见 Teaching-Dashboard 仓库。

## 关键文件

| 文件 | 说明 |
|------|------|
| `AGENTS.md` | 本文件 — AI 交接手册 |
| `README.md` | 人类可读总览，含目录结构和课表 |
| `备课规范.md` | 材料生成的完整格式规范（必读） |

## 目录结构（严格遵守）

```
{学生名}/
├── 学生/{周次}/
│   └── {编号}-{科目}-{内容}（学生版）.md     ← 学案，打印给学生
└── 教师/{周次}/
    └── {编号}-{科目}-{内容}（教师版）.md     ← 时间轴操作手册
```

### 核心原则：每节课每科只有1份学生版 + 1份教师版

- **学生版 = 学案**：好的题目 + 要点总结，不含任何答案，打印给学生做，课后带走当复习资料
- **教师版 = 时间轴操作手册**：题目+答案+解析+逐字稿+分支判断全部融入时间轴，老师只需要这一份文档

### 路径层级（固定 4 层，不可增减）

| 层 | 内容 | 示例 |
|----|------|------|
| 第1层 | 学生姓名 | `黄涵松` |
| 第2层 | `学生` 或 `教师` | `学生` |
| 第3层 | 两位数字周次 | `08` |
| 第4层 | 文件名 | `01-语文-语法与阅读理解（学生版）.md` |

> ⚠️ **前端网站 Teaching-Dashboard 的 `parseMaterialsTree()` 函数依赖此结构（`parts.length === 4`）。**
> 任何改动都会导致页面显示异常。这是一条全局红线，记录在 Teaching-System/AGENTS.md 中。

### 文件命名格式

```
{编号}-{科目}-{内容}（学生版）.md
{编号}-{科目}-{内容}（教师版）.md
```

| 元素 | 说明 | 示例 |
|------|------|------|
| 编号 | 两位数 01-99，按上课顺序 | 01、02、03 |
| 科目 | `语文` / `道法` / `历史` | 注意简称，不是"道德与法治" |
| 内容 | 具体知识点或本课主题 | 期中备考冲刺、语法与阅读理解 |
| 类型 | `（学生版）` / `（教师版）` | 括号包裹 |

### ❌ 严禁写法

```
学生/黄涵松/08/...          ← 学生名不能在第2层
黄涵松/学生/第8周/...        ← 周次必须纯数字，不加"第""周"
黄涵松/学生/08/学生版/...      ← 不可增加第4层子文件夹
黄涵松/08/...                ← 不可省略角色层
```

## 材料生成规范

详细规范见 `备课规范.md`，简要版如下：

| 类型 | 定位 | 核心内容 |
|------|------|----------|
| 学生版 | 学案 | 题目（小测+练习）+ 要点总结，不含答案 |
| 教师版 | 时间轴操作手册 | 按时间轴走：指令→题目→答案→解析→逐字稿→分支判断→练习讲评（方法融入）→课后安排 |

### 出题原则

**题目围绕下次考试出题。** 从教学日历确认下次考试范围，题目内容覆盖考试范围内的核心考点。

### 教师版标杆

`杨紫泠/教师/08/01-语文-期中备考冲刺（教师版）.md` — 所有教师版按此格式生成。

## 每次备课后的必须操作

1. 将所有生成的 `.md` 文件写入正确路径
2. 更新 `README.md` 中的目录结构和课表
3. 更新 `Student-Profiles` 中的学生档案和周次进度
4. 输出打印清单给教师确认

## 注意事项

- 分支为 `master`（非 main）— **这是全局红线**
- 所有文件为 Markdown 格式
- 科目用简称：`道法`（不是"道德与法治"）、`政治`（不是"思想政治"）
