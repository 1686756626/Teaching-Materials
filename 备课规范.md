# 备课材料生成规范

> 本文档是生成备课材料时的通用规则，适用于所有学生、所有科目、所有周次。

---

## 一、文件分类

### 学生用（`学生/`目录）
- 面向学生本人，语言直白、排版清晰
- 适合打印，纸质使用

### 教师用（`教师/`目录）
- 面向教师，逐字稿格式，照着就能讲
- 理解成本降到最低，不需要额外备课

> **两套材料互相独立，互不包含对方的内容。** 教师教案里不附参考答案，参考答案统一放在解析版中。

---

## 二、文件命名规范

| 后缀 | 含义 | 打印 | 目标读者 |
|------|------|:----:|---------|
| `_原卷.md` | 纯题目，无答案 | ✅ 打印 | 学生 |
| `_解析.md` | 答案+详细解析 | ❌ 不打印 | 教师 |
| `_要点.md` | 知识清单/知识梳理 | ✅ 打印 | 学生 |
| `_答题框架.md` | 答题模板/方法总结 | ✅ 打印 | 学生 |
| `_教案.md` | 逐字稿 | ❌ 不打印 | 教师 |

**命名结构：** `科目_内容_后缀.md`

**示例：**
- `语文_语法小测_原卷.md`
- `语文_语法小测_解析.md`
- `历史_古代史时间轴与考点.md`（无后缀 = 知识要点类）

---

## 三、原卷版规范

### 必须满足
- **无任何答案**，从头到尾纯题目
- 末尾留空白行，方便手写答案
- 顶部写明：姓名、日期、限时、满分
- 格式清晰，打印出来一目了然

### 不要出现
- ❌ 参考答案
- ❌ "打印时删除此部分"这类提示语
- ❌ 教师才能看到的内容

---

## 四、解析版规范

### 每道题必须包含

| 要素 | 说明 |
|------|------|
| **标准答案** | 明确写出选什么/填什么 |
| **详细解析** | 不只说"选C"，要说"为什么选C，为什么不选ABD" |
| **对比表格** | 涉及混淆知识点时，用表格对比（如：秦焚书坑儒 vs 汉独尊儒术） |
| **考点标注** | 标注该题对应的考点和重要程度（⭐~⭐⭐⭐⭐⭐） |
| **做题技巧** | 总结这类题的通用解法，帮助教师引导学生举一反三 |

### 格式要求
- 解析和原卷题目**一一对应**，方便教师快速翻看
- 用引用块（`>`）区分解析正文和补充说明
- 篇末附本测验的考点汇总表

### 语气
- 解析是写给**教师**看的，可以写"引导学生思考……""这时候你可以这样讲……"
- 不要写成面向学生的"你应该……"

---

## 五、要点/框架类材料规范

### 面向学生
- 语言简洁，用词贴近学生日常用语
- 多用表格、列表、编号，少用大段文字
- 重要内容加粗，核心公式/模板用引用块突出
- 末尾附"速记口诀"或"一句话总结"，方便记忆

### 内容要求
- **覆盖全面**但不面面俱到——挑高频考点
- 答题框架要给**公式**，不能只给思路（如"本义→语境义→效果"，不是"先想想词义再结合语境"）
- 知识梳理要有**结构**，按板块/单元/题型组织，不要平铺直叙

---

## 六、教案（逐字稿）规范

### 必须包含

| 要素 | 说明 |
|------|------|
| **头部信息** | 学生姓名、日期、时长、学校当前进度 |
| **课前准备** | 需要打印的文件清单（精确到文件名）、课前需要看的文件 |
| **时间分配** | 每个环节标注精确时间（如"0-5min""35-40min"） |
| **教师原话** | 用"你说：'……'"格式，写明教师实际要说的话 |
| **板书内容** | 用"你在纸上写：……"格式，写明要写什么 |
| **过渡语** | 环节之间的衔接话术，减少课堂冷场 |
| **分支判断** | 学生全对/错几道/错很多，分别怎么处理 |
| **课后布置** | 具体的作业内容和要求 |
| **教学记录表** | 末尾附表格，课后填写本节课情况 |

### 语气要求
- **口语化、大白话**，不要书面语
- 不要出现"教师应引导学生……"这种教案腔
- 要出现的是："涵松，来了？先别急，给你5分钟做一个小测"
- 让教师拿到就能直接讲，零备课成本

### 时间控制
- 一对一2小时课，总时长严格控制在120分钟内
- 每个环节的时间写清楚
- 预留弹性空间（5-10min），防止某个环节超时

## 七、文件联动检查

每次修改文件后，必须检查以下联动关系：

```
修改了 原卷/解析版 的文件名
  → 检查对应教案的【课前准备】引用
  → 检查 README 的目录结构

修改了 要点/框架 的内容
  → 检查对应教案中是否引用了具体知识点
  → 确保教案讲的和要点写的一致

新增了文件
  → 更新 README 目录结构
  → 检查是否需要配套新增其他文件（如新增原卷就要有解析版）

修改了 README
  → 确认文件列表和实际文件一一对应
```

**简单说：改一个文件，扫一遍所有可能引用它的地方。**

## 八、内容生成原则

### 三科共用2小时的取舍
- 每科约40分钟，**不可能讲太多**，一节课只聚焦1个核心知识点
- 小测5分钟摸底 → 核心15-20分钟精讲 → 抢进度10-15分钟 → 练习+总结
- 抢进度：挑最重要的1-2个考点速讲，不展开

### 知识点呈现方式
- **能用表格就不用文字**（对比记忆更高效）
- **能画结构图就不用列表**（知识脉络更清晰）
- **能给公式就不给思路**（学生直接套用，降低做题门槛）

### 简答题/分析题
- 必须给**答题框架**或**模板**，不能只列知识点
- 框架要具体到"先写什么、再写什么、最后写什么"
- 示例：散文句子赏析 = 判手法 → 套公式 → 联情感

### 易混淆点
- 用**对比表格**呈现，把相似概念放一起对比
- 标注"常见陷阱"，如"秦焚书坑儒 vs 汉独尊儒术"
- 帮教师预判学生可能搞混的地方

### 安全规范
- **Token、密码等敏感信息不入库**
- README 和教案中不写任何密钥
