# 备课材料生成规范

> 本文档是生成备课材料时的通用规则，适用于所有学生、所有科目、所有周次。

---

## 一、目录结构

```
{学生姓名}/
├── 学生/{周次}/     ← 打印给学生用的材料
└── 教师/{周次}/     ← 教师用材料
```

- 按**周次**分文件夹（格式如 `08`、`09`），与学校进度对应，不同周的材料自然隔开
- 日期文件夹内的文件按**上课顺序**编号

---

## 二、文件命名规则

```
编号-科目-内容（类型）.md
```

| 元素 | 说明 | 示例 |
|------|------|------|
| 编号 | 两位数，按上课顺序排 | `01`、`02`、`03` |
| 科目 | 中文 | `语文`、`道法`、`历史` |
| 内容 | 中文，具体知识点 | `语法小测`、`社会规则要点` |
| 类型 | 括号包裹 | `（原卷）`、`（解析）`、`（教案）` |

### 类型说明

| 后缀 | 含义 | 放哪个目录 | 打印 |
|------|------|:----------:|:----:|
| `（原卷）` | 纯题目，无答案 | 学生 | ✅ |
| `（解析）` | 答案 + 详细解析 | 教师 | ❌ |
| 无后缀 | 知识要点 / 答题框架 | 学生 | ✅ |
| `（教案）` | 逐字稿 | 教师 | ❌ |

### 编号规则

- 编号 = 上课顺序，跟时间轴对应
- 同一节课的教案和解析共享编号
- 学生用和教师用的编号互相对齐：

```
学生/04.05/
  01-语文-语法小测（原卷）.md        ← 配套 ↓
  02-语文-散文赏析答题框架.md
教师/04.05/
  01-语文-语法小测（解析）.md         ← 解析版，同编号
  01-语文-语法检验与散文框架（教案）.md ← 教案，同编号
```

---

## 三、原卷版规范

### 格式要求
- **绝对不能包含任何答案**
- 末尾留空白行
- 顶部写明：姓名、日期、限时、满分
- 格式清晰，打印出来一目了然

### 禁止出现
- ❌ 参考答案
- ❌ "打印时删除此部分"等提示
- ❌ 任何教师视角的内容

---

## 四、解析版规范

### 每道题必须包含

| 要素 | 说明 |
|------|------|
| 标准答案 | 明确写答案 |
| 详细解析 | 为什么选这个，不选别的 |
| 对比表格 | 涉及混淆点时用表格对比 |
| 考点标注 | 标注考点名称和重要程度 |
| 做题技巧 | 总结通用解法 |

### 格式要求
- 与原卷题目一一对应
- 用引用块（`>`）区分正文和补充说明
- 篇末附考点汇总表
- 语气面向教师，可写"你可以这样引导学生……"

---

## 五、要点 / 框架类规范

### 面向学生，要求
- 多用表格、列表，少用大段文字
- 答题框架给**公式**，不给模糊思路
- 重要内容加粗，核心模板用引用块突出
- 末尾附速记口诀

---

## 六、教案（逐字稿）规范

### 必须包含

| 要素 | 格式 |
|------|------|
| 头部信息 | 学生姓名、日期、时长、学校当前进度 |
| 课前准备 | 打印清单（精确文件名）+ 课前要看的文件 |
| 时间分配 | 精确到分钟（如 0-5min、35-40min） |
| 教师原话 | `你说："……"` |
| 板书内容 | `你在纸上写：……` |
| 过渡语 | 环节之间的衔接话术 |
| 分支判断 | 学生全对 / 错几道 / 错很多，分别怎么处理 |
| 课后布置 | 具体作业内容和要求 |
| 教学记录表 | 末尾附表格，课后填写 |

### 语气
- 口语化大白话，不要教案腔
- ❌ "教师应引导学生思考"
- ✅ "涵松，来了？先别急，给你5分钟做一个小测"

### 时间控制
- 总时长严格控制在120分钟内
- 预留5-10分钟弹性空间

---

## 七、文件联动检查

每次修改文件后，按以下清单检查：

| 改了什么 | 检查什么 |
|---------|---------|
| 文件名 | 教案课前准备中的引用、README 目录结构 |
| 文件内容 | 教案中引用的具体知识点是否一致 |
| 新增文件 | 配套文件是否齐全（有原卷→有解析） |
| README | 文件列表与实际文件是否一一对应 |

---

## 八、内容生成原则

### 三科共用2小时的取舍
- 每科约40分钟，只聚焦 **1个核心知识点**
- 结构：小测5min → 核心15-20min → 休息5min → 抢进度10-15min → 练习+总结

### 知识点呈现
- 能用表格就不用文字
- 能给公式就不给思路
- 易混淆点用对比表格
- 简答题给"先写什么→再写什么→最后写什么"的框架

### 安全
- Token、密码等敏感信息不入库
